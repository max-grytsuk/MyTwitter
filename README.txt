Серверная часть приложения.

Каркасом серверной части приложения является Zend Framework.

Используется принцип тонкого контроллера, максимум операций переносится в классы модели. 

Контроллер AuthController - отвечает за аутентификацию и регистрацию пользователей, а также за обработку запросов по изменению профиля пользователя. Валидация пользователя происходит в классе LoginClass.

Контроллер MainController - перенаправляет ajax запросы на обработку в класс MainClass, где происходит диспетчеризация этих запросов и перенаправление их в соответствующие классы-обертки таблиц базы данных (Users, Tweets, TweetersRelations).



Клиентская часть приложения.

В клиентской части приложения используются YUI3 и JQuery (в частности JQuery Tools (компоненты Validator, Tabs и Scrollable)).

Для ajax загрузки файлов используется библиотека Ajax Upload.

В качестве каркаса основной части приложения на клиенте принята модульная архитектура, предложенная Nicholas C. Zakas в лекции Scalable JavaScript Application Architecture. Суть ее заключается в использовании паттернов Наблюдатель и Медиатор.

В глобальном пространстве имен создаются два объекта: core и sandbox. 
Объект core отвечает за регистрацию и старт модулей. 
Объект sandbox является контейнером для хранения информации о всех запущенных модулях и о событиях, в которых эти модули заинтересованы. 

Каждый из запущенных модулей не знает о других модулях, а знает только о sandbox. При возникновении какого-либо события в нем, он оповещает об этом sandbox, а уже sandbox передает информацию об этом событии тем модулям, которые в нем заинтересованы. Этим достигается слабая связанность системы и однотипный интерфейс межмодульного взаимодействия.

Также у sandbox есть интерфейс для передачи ajax сообщений, однотипный для всех модулей и блок с функциями, общими для всех модулей.

В данном приложении существуют четыре модуля: 
1) tweets (отображение твитов);
2) user_panel (приведена информация о пользователе (аватар, мэйл) и списки твитеров, за которыми наблюдает пользователь и которые за ним наблюдают);
3) new_tweet_box (блок для добавления нового твита);
4) who_to_follow (список всех пользователей твитера);


P.S.
В репозитории отсутствуют файлы, отвечающие за конфигурацию приложения, а также файлы, связанные с оформлением (css, images).
